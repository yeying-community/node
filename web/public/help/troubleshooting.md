# 常见问题与排查

## 发布应用相关

### 创建了应用，为什么还不能接入登录？

应用创建后，还需要完成上架。  
只有已经上架的应用，才能用于正式登录。

### 登录后没有跳回应用怎么办？

先检查：

1. `redirectUri` 是否填写正确
2. 应用是否已经上架

### 提示 `redirectUri is not allowed` 是什么意思？

说明：当前回调地址不在允许范围内。

处理方法：

1. 回到应用中心检查配置的回调地址
2. 确认业务应用实际传入的地址完全一致

## 通行证登录相关

### 为什么无法注册通行证？

常见原因：

- 当前浏览器不支持通行证
- 站点没有开启通行证能力
- 当前环境配置还没准备好

### 为什么点击登录后没有弹出通行证界面？

常见原因：

- 当前用户没有注册通行证
- 浏览器不支持通行证
- 当前页面或域名配置不满足要求

### 为什么验证成功了，却没有完成登录？

通常优先检查：

1. 应用的 `AppId` 是否正确
2. `redirectUri` 是否正确
3. 应用是否已经上架

## 排查顺序

1. 先检查应用是否已上架
2. 再检查 `AppId`
3. 再检查 `redirectUri`
4. 最后检查通行证是否已注册

如果还是有问题，再查看浏览器控制台和网络请求。
