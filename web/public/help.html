Node 使用帮助

这里提供中心化 UCAN、应用发布与审批的最短操作路径。遇到鉴权错误时，优先核对 address、clientId、redirectUri。

快速开始（5 分钟）

  1. 使用钱包登录，进入应用中心。
  2. 打开“我的配置”，刷新 Mobile Auth 状态。
  3. 加载认证器配置，用 Authenticator 扫码绑定 TOTP。
  4. 填写客户端配置并创建授权请求。
  5. 批准授权并兑换 code，得到 JWT + UCAN。
  6. 在“结果验证”校验 Token 是否可用。

关键参数

  • clientId：建议使用应用市场 AppId（applications.uid）。
  • redirectUri：必须在该 clientId 的允许列表中。
  • audience：需与服务端 UCAN audience 一致。
  • capability.with/can：需匹配服务端策略，否则会能力不足。

常见错误处理

  • redirectUri is not allowed：回调地址未进入白名单。
  • UCAN audience mismatch：Token 的 aud 与服务端期望不一致。
  • UCAN capability denied：签发能力不足，需调整并重签。
  • USER_ROLE_DENIED：账号角色不满足发布/审批权限。

发布与审批建议

  • 发布前先确保应用基础信息和 location 配置正确。
  • 审批列表看不到任务时，先核对 approver 地址是否匹配当前登录地址。
  • 优先查看后端 logs/ 日志定位鉴权或角色问题。
