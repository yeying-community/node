# 产品使用手册（场景化）

本文档用于说明 Node 应用在产品侧如何使用。  
目标是把复杂能力拆成“按场景可执行”的步骤，减少理解成本。

## 适用对象
- 产品经理：理解功能边界与用户路径。
- 前端工程师：按统一参数接入认证与授权。
- 运营/测试：快速验证关键流程与常见故障。

## 功能地图
- 登录：钱包 SIWE 登录，拿到 JWT 会话。
- 授权：UCAN（钱包签发或中心化签发）。
- 审批：发布申请、使用申请、审批流转。
- 配置：`我的配置`页统一调试 Mobile Auth 与授权流程。

## 5 分钟上手（推荐）
1. 钱包登录 Node 应用，进入“应用中心”。
2. 打开“我的配置”，点击“刷新状态”，确认服务就绪。
3. 点击“加载认证器配置”，用 Authenticator 扫码绑定。
4. 在“授权配置”填写 `address/clientId/redirectUri`，点击“创建授权请求”。
5. 在“授权流程”完成 `TOTP 批准授权` 与 `兑换授权码`。
6. 在“结果验证”验证 `JWT` 和 `UCAN` 是否可用。

## 核心场景

### 场景一：发布应用
1. 进入“应用中心 -> 创建应用”。
2. 选择模板（Chat/Router/Warehouse）或自定义。
3. 完成基础信息并保存。
4. 提交上架申请，等待审批。

### 场景二：审批任务
1. 进入“我的审批 -> 待我审批”。
2. 查看“名称/申请人/状态/同意进度”。
3. 点击“去审批”提交同意或驳回意见。
4. 在“审批完成”通过“详情”查看申请详情和审批轨迹。

### 场景三：手机端无插件登录（中心化 UCAN）
1. PC 端钱包登录后，在“我的配置”绑定 TOTP。
2. 移动端应用提交授权请求并跳转 Node 验证页。
3. 用户输入 TOTP，Node 返回 `authorization code`。
4. 移动端用 `code + clientId + redirectUri` 兑换 JWT/UCAN。
5. 业务接口继续使用统一 Bearer 鉴权。

## 参数约定（强烈建议）
- `clientId`：建议直接使用应用市场 `AppId`（`applications.uid`）。
- `redirectUri`：必须进入允许列表，且与 `clientId` 匹配。
- `audience`：与服务端 `UCAN_AUD` 一致。
- `capability.with/can`：与服务端默认能力匹配，避免 capability denied。

## 常见错误与处理
- `redirectUri is not allowed`
  - 原因：`clientId + redirectUri` 未命中白名单。
  - 处理：修正 redirectUri，或把应用发布状态与 location 配置正确。

- `UCAN audience mismatch`
  - 原因：token 的 `aud` 与服务端期望不一致。
  - 处理：统一前后端 audience 配置。

- `UCAN capability denied`
  - 原因：token 能力不足。
  - 处理：对齐 `with/can` 配置并重新签发 token。

- `Admin permission required` 或 `Approver permission denied`
  - 原因：当前账号没有审批权限或不在 approver 列表。
  - 处理：检查审批单 approver 与当前登录地址是否匹配。

## 运维建议
- 先看 `logs/` 下接口日志，再看前端控制台错误。
- 优先核对三元组：`address`、`clientId`、`redirectUri`。
- 涉及授权问题时同时检查：`audience`、`capability`、token 有效期。
